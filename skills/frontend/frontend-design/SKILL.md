# Frontend Design

## When to Use

- Building landing pages, dashboards, or React components
- User asks to improve UI aesthetics or layout
- Creating marketing pages or web artifacts

## Design Thinking

Before coding, commit to a clear aesthetic direction:

- **Purpose**: Problem solved and target audience
- **Tone**: minimal, maximalist, retro, editorial, brutalist, etc.
- **Constraints**: framework, performance, accessibility
- **Differentiation**: one memorable visual hook

## Guidelines

- **Typography**: Avoid generic fonts (Arial, Inter). Use distinctive pairings.
- **Color**: Cohesive palette with CSS variables; strong accent colors
- **Motion**: High-impact page load or scroll animations; avoid scattered noise
- **Layout**: Asymmetry, overlap, or intentional grid-breaking where appropriate
- **Backgrounds**: Textures, gradients, or depth instead of flat defaults

## Anti-Patterns

Avoid generic AI aesthetics:

- Purple gradients on white
- Overused font stacks (Inter, Roboto, Space Grotesk everywhere)
- Cookie-cutter card grids with no context

## Instructions

1. Clarify purpose and constraints
2. Pick one bold aesthetic direction
3. Implement working code (React, HTML/CSS, etc.)
4. Refine spacing, typography, and motion details

Match implementation complexity to the vision: maximalist designs need richer code; minimal designs need precision and restraint.
